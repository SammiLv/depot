### 2.3 服务端能力：季度初始化 / 模板下载 / 模板导入

已在 `src/server/kpi/actions.ts` 落地：

#### 季度初始化
- 支持按当前范围批量初始化当季 `PersonalKpi`
- 按模板分配规则为成员匹配模板
- 自动复制 `KpiTemplateItem` 到 `PersonalKpiItem` 形成快照
- 已存在单据自动跳过
- 未命中模板成员跳过并返回结果摘要

#### 模板匹配规则
已支持以下分配优先级：
1. `USER`
2. `ORG_NODE`
3. `ROLE`

同时支持：
- 模板有效期判断
- 最新版模板选择
- 已审核通过模板优先

#### 模板下载
- `downloadKpiTemplateCsv`
- 已改为按部门下载模板文件
- 文件名带部门名

#### 模板导入
- `importKpiTemplates`
- 已改为按部门导入模板
- 支持一个 Excel 多 sheet 导入多个模板
- 导入后返回：
  - 成功导入模板数
  - 导入模板项数
  - 导入分配规则数
  - 导入模板名称列表
  - 目标部门名称

### 2.4 KPI 查询层改造

已在 `src/server/kpi/kpi-query.ts` 完成：

#### KPI 管理页基础数据
- 查询当前季度 KPI 数据

- `createdByName`
- `updatedByName`
- `scopeName`
- `createdAt`
- `updatedAt`
- `departmentOrgNodeId`

并按以下顺序倒序：
- `updatedAt desc`
- `createdAt desc`

## 5. 风险与注意事项

### 5.1 本地数据库漂移风险
这是当前分支最大的实际风险。

表现过的问题：
- Prisma Client 与 schema 不一致
- SQLite 实际表结构缺列
- 代码已查 `updatedById`，数据库表中没有对应字段
- 导入模板 / 查询模板时因为缺列直接报错

接手后建议第一时间执行：
1. `npm run prisma:generate`
2. 检查 migration 是否完整
3. 对照 `db/dev.db` 确认列是否存在
4. 再启动 `npm run dev:kpi`

### 5.2 工作区存在额外未整理文件
当前 `git status` 中除代码改动外，还有一些临时 / 新增文件：
- `kpi-department-import.xlsx`
- `kpi-template-import.xlsx`
- `requirements/handoff/kpi-function/`
- migration 目录
- `src/server/kpi/actions.ts`

需要提交前统一判断哪些应纳入版本管理，哪些应清理。

### 5.3 当前分支改动面较大
本轮不是单点修补，已经同时涉及：
- 数据模型
- 迁移
- seed
- 服务端 action
- 查询层
- 页面布局

后续继续改时要注意不要只修前端而漏掉 schema / 本地库 / Prisma Client。
