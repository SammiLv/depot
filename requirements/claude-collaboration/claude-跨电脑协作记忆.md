# Claude 跨电脑协作记忆

## 固定协作约定

### 1. 公司电脑运行方式
执行：

```bash
npm run dev:office
```

代表：

- 地址：`http://depot.rj-info.com/`
- 端口：`80`

### 2. 家里电脑运行方式
执行：

```bash
npm run dev:home
```

代表：

- 地址：`http://localhost:3000/`
- 端口：`3000`

### 3. 两台不同电脑可以同时运行
- 公司电脑跑 `80`
- 家里电脑跑 `3000`
- 互不冲突

只有同一台电脑、同一个仓库目录里同时启动两个 `next dev` 才会冲突。

### 4. 钉钉登录地址优先走 `APP_URL`
当前项目已经按环境变量区分：

- 公司：`APP_URL=http://depot.rj-info.com`
- 家里：`APP_URL=http://localhost:3000`

## 哪些内容适合写进这份文档
适合写：

- 长期稳定的协作规则
- 跨电脑仍然成立的启动方式
- 需要 Claude 长期遵守的工作约定

不适合写：

- 临时任务进度
- 某一次排查过程
- 很快会过时的状态

## 是否每次都要整理
不需要每次工作结束都整理。

只有在下面几种情况才需要补充或修改：

- 你确认了新的长期协作规则
- 启动方式变了
- 环境切换方式变了
- 有些习惯希望 Claude 以后一直记住

如果只是普通开发、修 bug、改页面，通常不需要更新这份文档。

## 推荐做法
把这份文档当成“长期稳定记忆”。

只有当你觉得某件事满足下面条件时，再更新：

- 下次换电脑还会用到
- 以后每次协作都应该遵守
- 不写下来以后容易重复解释
