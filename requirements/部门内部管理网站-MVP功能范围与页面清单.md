# 部门内部管理网站-MVP功能范围与页面清单（按当前实现更新）

## 1. 当前 MVP 判断
当前仓库已经完成从脚手架到 **可演示管理工作台** 的第一阶段搭建。

当前 MVP 的核心特征是：
- 有真实登录态与会话
- 有角色与菜单权限
- 有数据库模型和 seed 数据
- 有真实工作台数据查询
- 有年度指标与组织权限两大重点模块骨架
- 有待办、通知等个人工作入口
- 有钉钉登录与组织同步的预留集成点

## 2. 当前 MVP 已落地功能范围
### 2.1 登录与认证
已实现：
- 首页自动分流到登录页或工作台
- 模拟登录：选择用户 → 确认身份 → 登录
- 退出登录
- 钉钉授权登录入口
- 钉钉回调处理
- 首次钉钉用户自动创建或绑定本地用户

当前说明：
- 模拟登录可直接用于演示不同角色视角
- 钉钉链路已接代码，但仍需真实配置联调

### 2.2 角色、菜单与数据范围
已实现：
- 4 类角色：管理员 / 部门主管 / 组长 / 普通成员
- 菜单权限表
- 角色菜单权限映射
- 登录后按角色过滤左侧菜单
- 服务端按角色控制部分查询范围

### 2.3 工作台
已实现：
- 欢迎区与当前身份描述
- 年度指标完成度卡片
- 待我审批卡片
- 未完成待办卡片
- 风险预警卡片
- 年度指标进度区
- KPI 阶段分布区
- 季度工作完成情况区
- 我的待办区
- 最近动态区
- 未读通知提示

### 2.4 年度指标管理
已实现：
- 年度指标方案列表 / 内容展示
- 部门方案与小组方案维护入口
- 指标项维护
- 元指标维护
- 季度目标拆解维护
- 周进度 / 季度进度更新入口
- 风险状态与完成度展示
- 相关 server actions

### 2.5 组织与权限
已实现：
- 用户管理
- 小组管理
- 设置部门主管

### 2.6 个人工作入口
已实现：

## 8. 当前建议的 MVP 验收范围
建议将当前版本的 MVP 验收重点定义为：
- 能登录并区分不同角色视角
