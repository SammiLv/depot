# 部门内部管理网站 UI 页面设计建议方案

## 1. 设计定位

本系统建议采用现代明亮科技型 SaaS 管理后台风格，而不是传统 OA 风格。

目标观感：

```text
明亮
清爽
科技感
卡片化
数据化
轻量
专业
不花哨
```

避免观感：

```text
传统 OA
老式 ERP
厚重后台管理系统
深灰菜单
密集表格
大面积线框
按钮堆叠
页面压抑
```

整体风格可定义为：

```text
Bright Tech SaaS Dashboard
明亮科技型管理工作台
```

## 2. 视觉参考方向

可参考以下类型产品的视觉方向：

```text
飞书后台
Linear
Notion 数据库视图
现代 SaaS Dashboard
轻量数据中台
```

不建议参考：

```text
传统 OA 审批系统
老式 ERP 系统
老后台管理系统
```

## 3. 配色方案

推荐采用图二中的 RJ 组件色板，整体为浅色背景 + 白色卡片 + 蓝色主操作 + 蓝绿色辅助强调，不使用紫色作为主视觉，也不使用大面积复杂渐变。

### 3.1 品牌色

蓝色 blue：

```text
Primary-active 点击：#1E4ACE
Primary 常规：#3069F9
Primary-hover 悬浮：#5489FA
Primary-disabled 禁用：#C3DBFE
Primary-bg 浅色背景：#E8F2FF
```

蓝绿色 teal：

```text
Primary-active 点击：#088884
Primary 常规：#0C9C93
Primary-hover 悬浮：#2BB0A3
Primary-disabled 禁用：#B6E1DE
Primary-bg 浅色背景：#F3FCFB
```

### 3.2 中性色

线条 Line：

```text
分割线：#F0F0F0
边框：#D9D9D9
```

背景色 background：

```text
容器禁用背景：#EEEEEE
页面全局背景：#F5F5F5
容器背景：#FFFFFF
```

文字 Text：

```text
强调/正文标题：#181818
次强调/正文标题：#4B4B4B
次要信息/深：#777777
提示/置灰：#DDDDDD
纯白文字：#FFFFFF
```

### 3.3 功能色

```text
成功 Success：#00B42A，hover #4DCB6A，浅色背景 #E8FFEA
警告 Warning：#FF7D00，hover #FFA44D，浅色背景 #FFF7E8
错误 Danger：#F53F3F，hover #F87979，浅色背景 #FFECE8
信息 Info：#3069F9，hover #5489FA，浅色背景 #E8F2FF
```

使用原则：

```text
蓝色用于主操作、链接、当前菜单、审批中状态
蓝绿色用于辅助重点、数据图表第二主色、健康状态点缀
绿色用于完成/正常
橙色用于延期/预警
红色仅用于严重风险或驳回
中性色用于文字层级、边框、背景和禁用状态
```

## 4. 页面整体布局

采用经典但现代化的后台布局：

```text
顶部 Header
左侧轻量导航
中间内容区
右侧可选信息栏
```

结构示意：

```text
┌──────────────────────────────────────────────┐
│ 顶部：系统名 / 搜索 / 通知 / 用户             │
├──────────┬───────────────────────────────────┤
│ 左侧菜单 │ 页面标题 + 操作按钮                │
│          │                                   │
│          │ 数据卡片 / 图表 / 表格 / 待办      │
│          │                                   │
└──────────┴───────────────────────────────────┘
```

左侧菜单建议：

```text
白色或浅色背景
图标 + 菜单名
当前菜单使用 #E8F2FF 浅蓝底 + #3069F9 文字强调
菜单项间距充足
不要使用厚重深灰色侧边栏
```

## 5. 首页工作台设计

首页是系统第一印象，应体现科技感和效率感。

### 5.1 首页内容结构

建议包括：

```text
顶部欢迎区
关键指标卡片
待办审批
风险预警
年度指标进度
季度工作进度
KPI 进度
人才概览
```

示意：

```text
┌─────────────────────────────────────────────┐
│ 早上好，产品部工作台                         │
│ 本季度还有 27 天，3 项待审批，2 项风险预警     │
├─────────┬─────────┬─────────┬─────────┤
│ 年度完成 │ KPI进度 │ 待审批   │ 风险预警 │
│ 68.5%   │ 12/15   │ 3       │ 2       │
├───────────────────────┬─────────────────────┤
│ 年度指标完成趋势        │ 本周待办             │
├───────────────────────┼─────────────────────┤
│ 小组完成情况            │ 季度工作状态分布      │
└───────────────────────┴─────────────────────┘
```

### 5.2 首页视觉建议

```text
顶部欢迎区使用白色卡片或 #E8F2FF 浅蓝背景
指标卡片使用白底 + 彩色图标
图表主色使用蓝色 #3069F9，辅助使用蓝绿色 #0C9C93
预警卡片使用 #FFF7E8 柔和橙色背景
风险信息避免大面积刺眼红色
```

## 6. 功能页面设计

### 6.1 年度指标页面

建议采用“指标树 + 详情 + 进度”的组合，不要一开始就是大表格。

页面结构：

```text
指标总览卡片
左侧指标树
右侧指标详情
进度条
趋势图
周更新记录
审批状态
```

指标树层级：

```text
部门指标
  └─ 小组指标
      └─ 个人指标
          └─ 季度指标
```

适合展示：

```text
目标值
完成值
完成率
预期进度
风险状态
负责人
审批状态
最近更新时间
```

### 6.2 季度工作页面

建议采用“看板 + 列表 + 时间线”结合。

页面结构：

```text
季度重点工作：卡片列表
月度计划：时间线
周更新：动态流
延期事项：风险卡片
```

状态颜色：

```text
未开始：灰色
进行中：蓝色
已完成：绿色
延期完成：橙色
```

这样比传统工作计划表更直观。

### 6.3 KPI 页面

建议采用“流程进度条 + 评分表单 + 结果卡片”。

流程示意：

```text
草稿 → 组长审批 → 主管审批 → 自评 → 组长评 → 主管终评 → 完成
```

页面结构：

```text
KPI 基础信息
流程进度条
指标评分表单
自评说明
组长评价
主管评价
最终得分卡片
历史 KPI 趋势
```

视觉重点：

```text
减少传统审批表观感
突出流程状态
突出分数和评价结果
突出待办操作
```

### 6.4 人才发展页面

建议做成“员工成长档案”，不要做成 HR 表格。

人才画像详情页结构：

```text
头像 / 姓名 / 小组 / 岗位 / 入职时间 / 当前状态
KPI 趋势
人才盘点结果
晋升记录
奖励记录
合同与续签信息
预警提示
发展建议
```

适合使用：

```text
档案卡片
时间线
趋势图
标签
预警卡片
```

## 7. 通用组件风格

建议统一使用以下组件风格：

```text
圆角：12px - 16px
卡片：白底 + 轻阴影
按钮：圆角按钮
标签：彩色胶囊标签
进度条：蓝色或蓝绿色进度条
表格：轻边框、少线条
弹窗：居中、简洁
表单：分组卡片式
图标：线性简洁图标
```

避免：

```text
深灰粗边框
大面积纯黑内容区
密集表格线
过多红色
过多按钮
复杂渐变背景
紫色/青色渐变作为主视觉
过度装饰
```

## 8. 推荐前端 UI 技术

推荐：

```text
Next.js
+ Tailwind CSS
+ shadcn/ui
+ Recharts
+ Lucide Icons
```

原因：

```text
shadcn/ui 视觉更现代，比传统后台组件更清爽
Tailwind CSS 方便做明亮科技感
Recharts 足够支撑仪表盘和趋势图
Lucide Icons 图标风格简洁统一
Claude Code 容易基于这些组件生成和维护页面
```

如果优先快速搭建传统后台，也可以选择 Ant Design。但 Ant Design 默认观感更接近传统管理后台，需要额外美化才能避免 OA 感。

本项目更推荐：

```text
shadcn/ui
```

## 9. 通用页面模板

每个业务页面建议采用统一模板：

```text
1. 页面标题区
   - 标题
   - 一句话说明
   - 主操作按钮

2. 数据摘要区
   - 3 到 4 个关键指标卡片

3. 主内容区
   - 表格 / 看板 / 图表 / 时间线

4. 侧边信息区
   - 审批状态
   - 操作记录
   - 风险提示
```

这样可以保证系统整体风格统一。

第一版建议设计以下页面：

```text
1. 登录页
2. 首页工作台
3. 年度指标总览
4. 年度指标详情
5. 季度工作计划
6. 月度计划与周更新
7. KPI 模板管理
8. 个人 KPI 页面
9. KPI 审批/评分页面
10. 人才画像列表
11. 人才画像详情
12. 权限与组织管理
```

## 11. 页面体验原则

### 11.1 信息展示

```text
能用卡片表达的，不直接上大表格
能用状态标签表达的，不只用文字
能用进度条表达的，不只显示数字
能用趋势图表达的，不只显示历史记录
```

### 11.2 操作体验

```text
主操作按钮明显
次要操作弱化
危险操作需要确认
待办事项优先展示
审批动作尽量一步完成
```

### 11.3 数据体验

```text
部门主管看全局
组长看本组
成员看本人
首页优先展示当前用户最需要处理的事项
```

## 12. 最终建议

本项目 UI 方向建议确定为：

```text
现代明亮科技型 SaaS 管理后台
```

视觉实现上重点使用：

```text
浅灰页面背景 #F5F5F5
白色圆角卡片 #FFFFFF
蓝色主操作 #3069F9
蓝绿色辅助强调 #0C9C93
轻量图标
卡片化看板
状态标签
流程进度条
少边框表格
数据图表化
```

这样既能满足内部管理系统的专业性，也能避免传统 OA 的老旧感，适合产品部试点和后续公司内部推广。
