# 部门内部管理网站-年度指标业务规则详版（按当前实现更新）

## 1. 文档目的
本文档专门梳理“年度指标”模块当前已经落地的业务规则、页面行为、权限边界、统计口径与待补齐项。

说明：
- 本文档以当前代码实现为准，不把尚未落地的设想写成已实现规则。
- 本文档重点区分：**已实现规则**、**当前实现限制**、**后续待补齐规则**。

## 2. 模块定位
年度指标是当前系统完成度最高的业务模块之一，承担以下核心职责：
- 维护年度指标方案
- 维护指标项
- 维护元指标
- 维护季度目标拆解
- 更新季度/周进度
- 展示风险状态、完成度、版本与历史记录

当前模块更偏向“维护 + 展示 + 进度更新”的业务骨架，审批流与更复杂状态流转仍未完全闭环。

## 3. 当前数据模型对应关系
当前年度指标相关模型包括：
- `AnnualGoalPlan`：年度方案
- `AnnualGoalMetric`：年度指标项
- `AnnualGoalMetricSource`：元指标
- `AnnualGoalQuarterTarget`：季度目标
- `AnnualGoalProgress`：进度记录模型已预留
- `AnnualGoalRevisionLog`：修订日志模型已预留

当前核心层级关系如下：
1. 方案（Plan）
2. 指标项（Metric）
3. 元指标（Source，可选）
4. 季度目标（QuarterTarget）

说明：
- 部门方案可以直接维护指标项，并继续拆解元指标。
- 小组方案不直接维护元指标主数据，而是从本部门方案中选择指标项或元指标映射进本组方案。

## 4. 当前方案层规则
### 4.1 方案归属类型
当前仅支持两类方案归属：
- 部门方案（`DEPARTMENT`）
- 小组方案（`TEAM`）

### 4.2 方案字段
当前方案层已落地的主要字段包括：
- 年份
- 方案名称
- 说明
- 归属类型
- 所属部门
- 所属小组（仅小组方案需要）
- 版本号
- 是否激活
- 审批状态字段
- 修订原因
- 生效时间字段
- 删除时间（用于归档）

### 4.3 方案创建规则
当前已实现：
- 方案年份必填，且必须在合理范围内（2000~2100）
- 方案名称必填
- 方案必须明确归属为“部门”或“小组”
- 部门方案必须有所属部门
- 小组方案必须有所属小组

### 4.4 方案权限规则
当前只有以下角色可维护年度指标方案：
- 管理员（ADMIN）
- 部门主管（DEPARTMENT_MANAGER）

当前权限边界：
- 管理员可维护任意部门/小组方案
- 部门主管只能维护自己部门下的方案
- 组长、普通成员不能创建或编辑方案

### 4.5 方案归属校验规则
当前已实现：
- 部门主管创建部门方案时，部门归属自动受其本人部门范围限制
- 部门主管创建小组方案时，只能选择自己部门下的小组
- 不允许部门主管维护其他部门方案

## 5. 当前指标项规则
### 5.1 指标项基础字段
当前指标项已支持以下核心字段：
- 指标编码 `metricCode`
- 指标名称
- 指标说明
- 目标值 `targetValue`
- 当前值 `currentValue`
- 单位 `unit`
- 权重 `weight`
- 计算方式 `calculationType`
- 风险状态 `riskStatus`
- 排序值 `sortOrder`
- 调整时间 `adjustedAt`
- 进度更新时间 `progressUpdatedAt`

### 5.2 指标项计算方式
当前已支持的计算方式枚举包括：
- `RATIO`：比例型
- `BOOLEAN`：布尔型
- `MANUAL_SCORE`：手工评分型

说明：
- 当前页面和服务端已支持录入该字段。
- 但不同计算方式的差异化计算逻辑尚未完全展开，当前完成度主要仍按目标值/当前值进行展示与汇总。

### 5.3 指标项风险状态
当前已支持的风险状态包括：
- `NORMAL`：正常
- `SLIGHT_DELAY`：轻微滞后
- `RISK`：风险
- `COMPLETED`：已完成

说明：
- 风险状态当前可维护并展示。
- 风险状态尚未形成自动判定规则，当前主要依赖人工维护。

### 5.4 指标项权重规则
当前已实现：
- 指标权重必须为非负数
- 同一方案下所有未删除指标项的权重合计 **不能超过 100%**
- 创建和修改指标项时都会校验权重上限

说明：
- 当前规则是“不超过 100%”，不是“必须等于 100%”。
- 因此系统允许方案处于“权重未补满”的中间维护状态。

### 5.5 部门方案指标项规则
部门方案中的指标项当前支持：
- 直接新增
- 编辑
- 删除（逻辑删除）
- 维护权重、目标值、当前值、单位、计算方式、风险状态、排序

### 5.6 小组方案指标项规则
小组方案中的指标项当前不是完全自由录入，而是**从本部门方案中继承/映射**：
- 可以选择部门方案中的某个指标项
- 或选择部门指标项下的某个元指标
- 二选一，不可同时选择

当前已实现的约束：
- 小组方案只能选择本部门下的部门指标
- 小组方案不能引用其他部门的指标项或元指标
- 小组方案创建后，继承来源指标的名称、目标值、单位、计算方式、风险状态等信息
- 小组方案后续可调整的主要是：权重、排序

说明：
- 这符合当前代码里的“部门方案维护主数据，小组方案做选取映射”的实现方式。
- 当前并未实现“小组方案可随意脱离部门方案独立改写业务口径”。

## 6. 当前元指标规则
### 6.1 元指标定位
元指标是挂在某个指标项下的更细粒度拆解项，用于承接最细目标值和季度拆解。

### 6.2 元指标基础字段
当前元指标支持：
- 元指标编码
- 名称
- 说明
- 目标值
- 当前值
- 单位
- 计算方式
- 风险状态
- 调整时间
- 进度更新时间

### 6.3 元指标创建范围
当前只有部门方案中的指标项可以继续拆解元指标。

当前页面行为也已体现：
- 小组方案不单独维护元指标主数据
- 小组方案是选择部门指标或部门元指标进行承接

### 6.4 元指标目标值约束
当前已实现：
- 同一指标项下，所有元指标目标值之和 **不得大于** 该指标项的目标总额

说明：
- 当前规则仍是“不得大于”，不是“必须完全等于”。
- 因此允许指标项暂时只完成部分元指标拆解。

### 6.5 元指标删除影响
当前已实现：
- 删除元指标时，已分配到小组的同源指标也会同步删除

这意味着：
- 部门方案中的元指标是小组方案引用关系的上游主数据
- 删除上游元指标会同步清理下游映射指标

## 7. 当前季度目标规则
### 7.1 季度目标定位
季度目标用于把年度指标项或元指标拆分到 Q1~Q4。

### 7.2 季度目标适用对象
当前季度目标可以挂在：
- 指标项上
- 元指标上

### 7.3 季度目标字段
当前季度目标已支持：
- 年份
- 季度（Q1~Q4）
- 季度目标值
- 季度当前值
- 本周新增值
- 风险状态
- 调整时间
- 进度更新时间

### 7.4 季度目标拆解规则
当前已实现：
- 每次保存季度目标时，以 Q1~Q4 表单为准整体重写
- 原有未删除季度目标会先逻辑删除，再按新值重建
- 季度目标值、季度当前值都不能小于 0

### 7.5 季度目标总额约束
当前已实现：
- 某指标项的 4 个季度目标值总和，不得大于该指标项目标总额
- 某元指标的 4 个季度目标值总和，不得大于该元指标目标总额

说明：
- 当前仍允许“季度目标未拆满全年目标”的情况
- 即规则是“不得超出”，不是“必须完全拆满”

### 7.6 小组方案季度目标显示规则
当前查询逻辑已支持：
- 小组方案如果是从部门指标或部门元指标继承而来，可继承对应的季度目标展示

这意味着当前页面上：
- 小组方案可以沿用部门目标拆解结果作为展示基础
- 但业务上还未形成完全独立的小组季度拆解与审批闭环

## 8. 当前进度更新规则
### 8.1 季度进度更新
当前已实现：
- 可按季度更新季度目标的当前值
- 当前值不能小于 0
- 更新后会同步刷新“进度更新时间”
- 更新后会自动回写并重算上层当前值

### 8.2 周进度更新
当前已实现：
- 仅针对“当前季度”的季度目标做周更新
- 可填写“本周新增”与“本季度当前值”
- 数值不能小于 0
- 批量更新时会校验重复项与目标归属
- 更新后会同步回写并重算上层当前值

### 8.3 当前值同步重算规则
当前已实现：
- 若更新的是元指标季度目标，则先汇总到元指标 `currentValue`
- 若指标项本身直接有季度目标，则指标项 `currentValue` 取其季度当前值汇总
- 若指标项没有直接季度目标，但有元指标，则指标项 `currentValue` 汇总自各元指标 `currentValue`

说明：
- 当前系统已经具备“季度进度 → 元指标/指标项当前值联动”的自动同步能力。
- 这也是当前年度指标模块较完整的部分之一。

## 9. 当前完成度与统计口径
### 9.1 单项完成度
当前展示层完成度主要按以下口径计算：
- `currentValue / targetValue`
- 结果按百分比展示

### 9.2 方案加权完成度
当前方案完成度为加权汇总：
- 先按每个指标项的完成度乘以该项权重
- 再按方案总权重汇总
- 页面展示为 `weightedProgress`

### 9.3 总体完成度范围
当前汇总口径更偏向部门年度指标总览：
- `overallWeightedProgress` 只统计部门方案中的指标项
- 小组方案更多用于部门拆解与视图展示，不直接并入部门整体汇总口径

### 9.4 风险数与修订数
当前摘要统计还包括：
- 风险指标数量：`riskStatus = RISK` 的指标数
- 修订计数：带 `revisionReason` 的方案数

## 10. 当前历史记录与版本规则
### 10.1 归档机制
当前已实现：
- 方案可归档
- 归档后从当前列表移出，进入历史记录
- 归档本质上通过 `deletedAt` 与 `isActive` 实现

### 10.2 历史记录展示
当前页面已支持：
- 查看历史记录列表
- 查看历史方案详情
- 查看归档时间
- 恢复历史方案到当前列表

### 10.3 版本字段现状
当前模型中存在：
- `version`
- `revisionReason`
- `revisedFromPlanId`
- `AnnualGoalRevisionLog`

但当前实际实现状态是：
- 页面已展示版本与历史记录能力
- 已有归档 / 恢复操作
- 但尚未形成完整的“修订生成新版本 + 自动写修订日志 + 审批通过生效”的正式版本流转机制

## 11. 当前权限与数据范围规则
### 11.1 查看范围
当前年度指标数据查询会按登录用户数据范围过滤：
- 管理员：可看全部
- 部门主管：可看本部门
- 组长：可看本组相关
- 普通成员：受当前数据范围规则限制

### 11.2 维护范围
当前能维护年度指标的角色只有：
- 管理员
- 部门主管

### 11.3 进度更新范围
当前季度进度更新与周进度更新范围更宽一些：
- 管理员可更新
- 对本部门方案有权限的部门主管可更新
- 若当前用户属于某小组，且该小组已承接该部门指标/元指标，则该小组范围内允许更新对应季度指标

说明：
- 这部分体现出“维护方案”和“更新季度进度”是两套权限边界。
- 方案主数据维护仍然更集中在管理员/部门主管。

## 12. 当前实现限制
以下规则或能力目前尚未完全闭环：
- 指标审批流未真正跑通
- 方案审批状态虽有字段，但未形成完整提审/审批动作闭环
- 修订日志模型已存在，但未形成完整自动落库链路
- 不同计算方式的专属统计逻辑未完全展开
- 风险状态未自动计算，当前主要靠人工维护
- 权重、元指标、季度目标目前只校验“不超额”，未强制“必须补满”
- `AnnualGoalProgress` 进度记录模型已预留，但当前更新动作主要直接写季度目标与当前值，没有形成完整进度台账页面

## 13. 建议补齐的下一阶段规则
建议下一阶段优先补齐以下年度指标规则：
1. 方案提审、审批、驳回、通过的完整状态流转
2. 版本修订时自动生成新版本与修订日志
3. 风险状态自动判定规则
4. 不同计算方式的差异化完成度算法
5. 季度目标“必须拆满”与权重“必须补足”是否需要变成强校验
6. 进度更新历史台账与审计记录
7. 小组方案与部门方案之间更明确的联动/冻结规则

## 14. 当前结论
如果以“是否已经具备可维护的年度指标主流程”为标准，当前答案是：**是**。

如果以“是否已经具备完整正式业务闭环”为标准，当前答案是：**还没有**。

当前年度指标模块已经实现了：
- 方案维护
- 指标维护
- 元指标拆解
- 季度拆解
- 周/季度进度更新
- 权限控制
- 历史记录展示
- 加权完成度统计

但尚未完全实现：
- 正式审批流
- 完整版本修订机制
- 自动风险判定
- 进度台账闭环
