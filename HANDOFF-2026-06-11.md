## 当前第一 blocker

### annual-goals 页面运行时报错
报错内容：
- `Runtime TypeError: Cannot read properties of undefined (reading 'get')`

当前判断：
- 这个问题**高概率出在 `src/app/(authenticated)/annual-goals/content.tsx` 的表单提交方式混用**
- 不是数据库结构问题
- 也不像是页面列表数据问题

今天已尝试过的方向：
- 修 `key` warning（已解决，但不是这个 runtime error 的根因）
- 把部分表单从内联 `action={async (fd) => ...}` 改成 `onSubmit + new FormData(event.currentTarget)`
- 改过一些删除/恢复/归档类按钮提交方式
- 怀疑点一直集中在：**同一页面里混用了多套表单提交模式**，导致某个路径里传入 server action 的 `FormData` 实际是 `undefined`

当前最值得继续查的重点：
- 全量检查 `src/app/(authenticated)/annual-goals/content.tsx`：
  - 是否还存在 `action={async (fd) => ...}`
  - 是否还存在把 server action 函数直接塞给客户端 onSubmit 包装、但调用时参数不对的情况
  - 是否还存在 `submitWithClose(...)` 之类中间封装导致 `FormData` 丢失
- 全量 grep `fd.get(` 和 `formData.get(` 的调用链
- 不要再泛泛怀疑数据问题，优先锁定“哪个表单 / 哪个 action / 哪一条触发栈把 `FormData` 传丢了”

一句话：
> **明天第一优先级不是继续抽象 org tree，而是先专杀 annual-goals 这个 `undefined.get` runtime error。**
